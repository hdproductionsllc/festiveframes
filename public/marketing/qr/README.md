# Booth QR Codes

Generated by `scripts/generate-qr.mjs` (uses the `qrcode` npm package).
Re-run with `node scripts/generate-qr.mjs` to regenerate.

All three codes are 1024px square PNGs, navy `#0F1B33` on white, with
high error correction (level H) and a wide quiet zone so they scan
reliably from a few feet away and survive print wear.

## What each code points to

Every code lands on the `/buy` page with a UTM campaign tag so we can
see which physical placement drove the scan.

| File            | Destination                                            | Use it on            |
| --------------- | ------------------------------------------------------ | -------------------- |
| `buy-booth.png` | `/buy?utm_source=qr&utm_campaign=booth`                | Booth signage, A-frame, table placard |
| `buy-car.png`   | `/buy?utm_source=qr&utm_campaign=car`                  | Car decal / window cling on the demo vehicle |
| `buy-card.png`  | `/buy?utm_source=qr&utm_campaign=card`                 | Business cards, flyers, takeaway handouts |

Base domain comes from `SITE_URL` in `src/config/season.ts`.

## Minimum print size (so it scans from 3-6 feet)

Rule of thumb: a QR code scans cleanly at a distance of about 10x its
printed width. For a typical phone camera held at arm's length:

| Scan distance | Minimum printed QR size |
| ------------- | ----------------------- |
| Up close (1-2 ft, handout/card) | 0.8 in (about 2 cm) |
| 3 ft (table placard) | 3.5 in (about 9 cm) |
| 6 ft (A-frame, banner) | 7 in (about 18 cm) |

When in doubt, print bigger. Keep the white quiet zone around the code
clear (do not crop into it or place text against the edge), and keep
the navy-on-white contrast (do not recolor the code or print it on a
busy background).

The PNGs are 1024px, which holds up sharp at 300 dpi up to about
3.4 inches. For anything larger than that (A-frame, banner) bump the
`width` in `scripts/generate-qr.mjs` and regenerate so the print stays
crisp.
