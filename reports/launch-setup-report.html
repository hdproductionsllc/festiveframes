3 · Go-Live Activation Sequence

  1. Point festiveframes.co DNS at Railway & add the custom domain to the Railway service. The keystone — start here. DNS currently at GoDaddy, serving a parked /lander.
  2. Revert SITE_URL in src/config/season.ts back to https://festiveframes.co, commit & push → Railway auto-redeploys. Fixes canonical / OG / sitemap.
  3. Set Railway env vars: STRIPE_WEBHOOK_SECRET, RESEND_API_KEY, EMAIL_FROM, ADMIN_ORDER_EMAIL, SUBSCRIBE_ENDPOINT. STRIPE_SECRET_KEY is already set; checkout is live.
  4. Register the Stripe webhook → endpoint /api/stripe/webhook, event checkout.session.completed; paste its whsec_… into STRIPE_WEBHOOK_SECRET. This is what actually turns order emails ON.
  5. Verify the sending domain in Resend (add DKIM/SPF DNS records) so customer emails reach real buyers. Until verified, Resend only sends to your own address.
  6. Set up email receiving (MX) for hello@festiveframes.co via Google Workspace / Zoho / Fastmail; add MX + SPF (+ DKIM/DMARC).
  7. Add festiveframes.co to Stripe → Settings → Payment method domains, to enable Apple/Google Pay.
  8. Live test: place one real order → confirm /thanks renders AND both the customer confirmation and your admin order email arrive.

4 · Recommendation

Do steps 1 and 2 before any promotion. The line between "can take an order" and "ready to drive traffic" is almost entirely the domain pointing and order emails. Everything else is safe to finish in the days after.

The full sequence above is captured in the go-live runbook. When the domain is pointed at Railway, the remaining work is dashboard config + Railway env vars — each Railway change auto-redeploys, so no further code is required to launch.
